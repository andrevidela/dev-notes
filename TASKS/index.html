    • [-] Reproduce bug about namespaces and non-total functions at typechecking
      • NOTE: If you implement a function f inside a module NS and you create an additional namespace NS inside your module and have another function f inside it. Then implementing the nested f (NS.NS.f) as f = NS.f won't refer to the f at the top level module, but will refer to itself. This is obviously not total, and if you call NS.NS.f then it will loop forever. If NS.NS.f happens to occur at compile-time, the typechecker will hang forever.
